# Output Directory

This folder contains generated records for the coupled 2010 electricity, occupancy, weather, building, and heating/cooling dataset.

## Records

| Record | File layout                                                 | Columns |
| --- |-------------------------------------------------------------| --- |
| E | `E.parquet`                                                 | `timestamp`, `profile_id`, `electricity_demand` |
| O | `O.parquet`                                                 | `timestamp`, `profile_id`, `occupied` |
| W | `W/loc{location_id:04d}.parquet`                            | `timestamp`, `air_temperature`, `relative_humidity`, `global_horizontal_irradiance`, `direct_normal_irradiance`, `diffuse_horizontal_irradiance`, `wind_speed` |
| B | `B.parquet`                                                 | `archetype_id`, `construction_year`, `area_m2`, `n_floors`, `height_floor_m`, `thermal_resistance`, `thermal_capacitance`, `window_area_total_m2` |
| H/C | `HC/loc{location_id:04d}/hc_arch{archetype_id:02d}.parquet` | `timestamp`, `profile_id`, `q_heat_w`, `q_cool_w` |

For records W and H/C, `location_id` (and `archetype_id` for H/C) are encoded in the filename rather than as columns inside the file.

## Identifiers

- `profile_id`: integer 1-74, assigned from sorted HTW Berlin profile filenames.
- `location_id`: integer 1-4045, assigned from the German grid centroid order.
- `archetype_id`: integer 1-11, assigned from the German TABULA archetype table.

The three mapping CSVs below are regenerated by the pipeline and are not tracked in git; a fresh clone will not contain them until the pipeline runs.

`profile_mapping.csv` stores the mapping from `profile_id` to the original electricity filename and annual demand (`source_file`, `annual_demand_kwh`).

`location_mapping.csv` stores the mapping from `location_id` to `latitude` and `longitude` (EPSG:4326). Generated by `build-location-mapping` or as a side effect of `fetch-weather`.

`archetype_mapping.csv` stores the mapping from `archetype_id` to the input archetype properties (`construction_year`, `area_m2`, `n_floors`, `height_floor_m`). Generated by `build-archetype-mapping` or as a side effect of `build-archetypes`.
